# Overview

요즘 핫한 AI 메타인 OpenClaw를 설치하고 슬랙을 통해 이런 저런 명령을 수행할 수 있도록 설정해보았다. OpenClaw는 권한만 주면 내 PC를 제어할 수 있기 때문에 보안에 취약할 수 있어, 격리된 가상머신 환경에서 구축을 진행했다.

<br />
<br />

# 환경

* **OS**: macOS m4 pro
* **VM**: Ubuntu 24.04.4 (UTM)

<br />
<br />

# Getting Started

## 1. UTM 설치 및 VM 구성하기

가장 먼저 격리된 환경에서 에이전트를 구동하기 위해 UTM을 설정한다.

### 사전 준비물

- **UTM**: https://github.com/utmapp/UTM/releases
- **Ubuntu 24.04.4 (AMD64)**: https://ubuntu.com/download/desktop

### VM 생성

- **Ubuntu Server** 또는 **Ubuntu Desktop(GUI 권장)** 으로 설치한다.
- ARM 기반 Mac의 경우 Ubuntu ARM 이미지를 사용한다.

### 시스템 최적화

### 클립보드 공유

VM과 호스트 간 클립보드를 공유하려면 다음을 설치하고 UTM 설정에서 **Clipboard Sharing**을 활성화한다.

```bash
sudo apt install spice-vdagent
```

<br />

### 화면 잠금 해제

에이전트의 연속 실행을 위해 자동 잠금을 비활성화한다.

```bash
gsettings set org.gnome.desktop.session idle-delay 0
gsettings set org.gnome.desktop.screensaver lock-enabled false
```

## 2. Slack 워크스페이스 및 앱 설정

OpenClaw 설치 전에 슬랙에서 워크스페이스를 만들고 앱을 준비해 둔다. 이 단계에서 발급받은 토큰들은 이후 OpenClaw 설정에서 사용한다.

### 워크스페이스 생성

- [slack.com](https://slack.com)에서 새 워크스페이스를 생성한다. 기존 워크스페이스가 있다면 건너뛰어도 된다.

### Slack 앱 생성 및 토큰 발급

1. [api.slack.com/apps](https://api.slack.com/apps)에 접속해 **Create New App** → **From scratch** 로 앱을 생성한다.
2. **Socket Mode** 를 활성화한다.
3. **Basic Information** → **App-Level Tokens** → **Generate Token and Scopes** 에서 `connections:write` 권한을 추가하고 **App Token** (`xapp-...`) 을 복사한다.
4. **OAuth & Permissions** 에서 Bot Token Scopes 를 추가한다.

   **필수 최소** (DM + 채널 기본 대화):
   - `chat:write`, `channels:history`, `channels:read`, `groups:history`
   - `im:history`, `im:read`, `im:write`
   - `users:read`, `app_mentions:read`

   **선택 추가** (아래 기능이 필요할 때만):
   - `groups:read`, `groups:write`, `mpim:history`, `mpim:read`, `mpim:write` - 그룹 DM·비공개 채널 쓰기
   - `reactions:read`, `reactions:write`, `pins:read`, `pins:write`, `emoji:read` - 반응, 고정, 이모지
   - `assistant:write` - 스트리밍 시 타이핑 표시
   - `commands`, `files:read`, `files:write` - 슬래시 명령, 파일 첨부
5. **Install to Workspace** 를 실행한 뒤 **Bot User OAuth Token** (`xoxb-...`) 을 복사한다.
6. **Event Subscriptions** 에서 Subscribe to bot events 를 설정한다. 이벤트를 추가하면 Slack 이 필요한 OAuth 스코프를 자동으로 붙여준다.

   **필수 최소** (DM + 채널 기본 대화):
   - `app_mention` - 봇 멘션 메시지
   - `message.channels` - 공개 채널 메시지
   - `message.groups` - 비공개 채널 메시지
   - `message.im` - 다이렉트 메시지
   - **App Home** → **Messages Tab** 활성화 (DM 용)

   **선택 추가** (아래 기능이 필요할 때만):
   - `message.mpim` - 그룹 DM
   - `reaction_added`, `reaction_removed` - 반응 이벤트
   - `member_joined_channel`, `member_left_channel` - 채널 멤버 변경
   - `channel_rename` - 채널 이름 변경
   - `pin_added`, `pin_removed` - 고정 메시지

자세한 설정은 [OpenClaw Slack 문서](https://docs.openclaw.ai/channels/slack)를 참고한다.

## 3. Claude API 키 발급

OpenClaw가 AI 모델을 사용하려면 Anthropic Claude API 키가 필요하다. 이 단계는 OpenClaw 온보딩 전에 미리 진행해 두면 편하다.

만약 다른 모델을 사용하고 싶다면 해당 모델의 API를 결제해서 사용하면 된다.

### Anthropic 콘솔 가입

1. [console.anthropic.com](https://console.anthropic.com)에 접속한다. (Claude.ai와는 별도의 개발자 콘솔이다.)
2. 회원가입 후 이메일 인증을 완료한다.

### 결제 및 크레딧 설정

1. **Billing** 섹션에서 결제 수단(카드)을 등록한다.
2. Claude API는 사용량 기반 과금(pay-per-use)이다. 사용한 토큰만큼 과금된다.
3. 필요하면 **Spending limit** 으로 예산 상한을 설정할 수 있다.

### API 키 생성

1. **API Keys** ([console.anthropic.com/settings/keys](https://console.anthropic.com/settings/keys)) 로 이동한다.
2. **Create Key** 를 눌러 새 API 키를 생성한다.
3. 생성된 키(`sk-ant-...`)를 안전한 곳에 보관한다. 한 번만 표시되므로 유의한다.

## 4. OpenClaw 설치

공식 CLI를 사용하여 에이전트 엔진을 설치하고 시스템 서비스로 등록한다.

### CLI 설치

[공식 설치 가이드](https://docs.openclaw.ai/install)에 따라 `curl` 등을 이용해 설치를 진행한다.

<br />

### 온보딩 (onboard)

설치 후 `openclaw onboard` 를 실행하면 설정 마법사가 진행된다. 2·3단계에서 준비한 값을 입력한다.

- **Model auth**: Claude API 키 (또는 `ANTHROPIC_API_KEY` 환경 변수)
- **Channels**: Slack App Token (`xapp-...`), Bot Token (`xoxb-...`)
- **Gateway**: 포트, 바인드, 인증 설정
- **Daemon 설치**: 마지막에 `--install-daemon` 옵션으로 서비스 등록 여부 선택

```bash
openclaw onboard
# 대화형으로 진행 후, Daemon 설치를 선택하거나 아래처럼 한 번에:
openclaw onboard --install-daemon
```

<br />

### onboard Step by Step

`openclaw onboard` 명령어를 실행하면 다음과 같은 대화형 마법사가 진행된다.

**Step 1. Security** - 보안 경고 화면이 나타난다. OpenClaw가 베타 단계이며 파일 읽기·도구 실행 등으로 예상치 못한 위험이 있을 수 있다는 내용을 읽고, Pairing/allowlists, Sandbox, 비밀정보 보호 등 권장 기준선을 확인한다. `openclaw security audit --deep` 등 보안 감사 명령어와 문서 링크가 안내된다. 이해했다면 **Yes**를 선택해 진행한다.

![onboard-01](/images/blog-posts/15-가상환경에-open-claw-설치해보기/openclaw-onboard-01.png)

**Step 2. Onboarding mode** - **QuickStart** 또는 **Manual** 중 하나를 선택한다. QuickStart는 기본 설정으로 빠르게 진행하고, 이후 `openclaw configure`로 세부 설정을 변경할 수 있다. 처음에는 QuickStart를 권장한다.

![onboard-02](/images/blog-posts/15-가상환경에-open-claw-설치해보기/openclaw-onboard-02.png)

**Step 3. Model/auth provider** - 사용할 AI 모델 제공자를 선택한다. Anthropic(Claude), OpenAI, Google, xAI, Mistral 등 다양한 제공자 중에서 고를 수 있다. 3단계에서 준비한 Claude API 키를 쓸 예정이라면 **Anthropic (setup-token + API key)**를 선택한다.

![onboard-03](/images/blog-posts/15-가상환경에-open-claw-설치해보기/openclaw-onboard-03.png)

**Step 4. Anthropic API key** - Anthropic API 키 입력 프롬프트가 나온다. 3단계에서 발급한 API 키(`sk-ant-...`)를 붙여넣는다.

![onboard-04](/images/blog-posts/15-가상환경에-open-claw-설치해보기/openclaw-onboard-04.png)

**Step 5. Default model** - 기본으로 사용할 Claude 모델을 선택한다. Haiku, Sonnet, Opus 등 다양한 모델이 있으며, 비용·성능에 맞게 선택한다. 예를 들어 `anthropic/claude-haiku-4-5-20251001`처럼 가벼운 모델부터 시작하는 것이 좋다. 원하는 모델이 목록에 없으면 **Enter model manually**로 직접 입력할 수 있다.

![onboard-05](/images/blog-posts/15-가상환경에-open-claw-설치해보기/openclaw-onboard-05.png)

**Step 6. Select channel** - 메시지를 주고받을 채널을 선택한다. Slack, Telegram, Discord, Google Chat 등이 있다. 2단계에서 준비한 Slack 토큰을 사용할 경우 **Slack (Socket Mode)**를 선택하면 된다. 이미 토큰을 설정했다면 **(configured)**로 표시된다.

![onboard-06](/images/blog-posts/15-가상환경에-open-claw-설치해보기/openclaw-onboard-06.png)

**Step 7. Configure skills** - 스킬(Skills)을 지금 설정할지 묻는다. 스킬은 에이전트의 기능을 확장하는 역할을 한다. 추후에 설정해도 되므로, 빠르게 넘어가려면 **No**를 선택해도 된다.

![onboard-07](/images/blog-posts/15-가상환경에-open-claw-설치해보기/openclaw-onboard-07.png)

**Step 8. Enable hooks** - 추가 훅(Hooks)을 활성화할지 선택한다. **command-logger**는 모든 명령 이벤트를 감사 파일에 기록하고, **session-memory**는 `/new` 또는 `/reset` 시 세션 컨텍스트를 메모리에 저장한다. 운영·디버깅에 유용하므로 둘 다 활성화해 두는 것을 권장한다.

![onboard-08](/images/blog-posts/15-가상환경에-open-claw-설치해보기/openclaw-onboard-08.png)

<br />

### Daemon 등록

온보딩에서 Daemon 설치를 선택했거나 `--install-daemon` 으로 진행했다면, systemd 를 리로드하고 서비스를 활성화한다.

```bash
sudo systemctl daemon-reload
sudo systemctl enable openclaw
sudo systemctl start openclaw
```

## 5. OpenClaw 설정 (Slack 연동)

슬랙을 통해 에이전트와 대화하기 위한 핵심 설정 단계이다.

슬랙에서 메세지를 보내면 다음과 같은 메세지가 출력된다. 이 때 페어링 코드를 받아 승인을 해줘야 한다.

<br />

```bash
OpenClaw: access not configured.

Your Slack user id: U07...

Pairing code: [PAIRING_CODE]

Ask the bot owner to approve with:
openclaw pairing approve slack [PAIRING_CODE]
```

### Pairing

슬랙 유저 ID 승인 절차를 거쳐 보안 관리자로 등록한다.

```bash
openclaw pairing approve slack [PAIRING_CODE]
```

### Channel Policy

모든 채널에서 자유롭게 대화할 수 있도록 **Open (allow all channels)** 설정을 권장한다.

### Threading

채널이 지저분해지지 않도록 모든 답변을 스레드로 고정한다.

```bash
openclaw config set channels.slack.replyToMode all
```

## 6. OpenClaw Browser 설정하기

에이전트가 브라우저를 조작할 수 있도록 연결한다.

### Chrome 설치

Ubuntu VM 에는 기본적으로 Chrome 이 없으므로 먼저 설치한다.

```bash
sudo apt update

# Chrome deb 패키지 다운로드 및 설치 (AMD64)
wget https://dl.google.com/linux/direct/google-chrome-stable_current_amd64.deb
sudo dpkg -i google-chrome-stable_current_amd64.deb
```

ARM 기반 VM 의 경우 [Chrome for Linux](https://www.google.com/chrome/browser/desktop/) 에서 해당 아키텍처용 패키지를 받아 설치한다.

설치 후 Chrome 을 한 번 실행해 기본 설정과 프로필 생성을 완료한다.

### 프로필 설정

아무런 설정을 하지 않으면 기본값인 `chrome` 프로필을 사용하여 브라우저 확장 프로그램과 연동한다.

```bash
openclaw config set browser.enabled true
```

### Auth Token

`openclaw.json`의 `gateway.auth.token` 토큰을 브라우저 확장 프로그램 설정창에 정확히 입력해야 `mismatch` 에러를 피할 수 있다.

## 7. openclaw.json 설정

설정에 어려움을 겪는 이들을 위해 예시 설정을 제공한다.

### 설정 파일 적용하기

아래 예시를 복사해 바로 적용할 수 있다.

1. **설정 디렉터리 생성** (없는 경우)
   ```bash
   mkdir -p ~/.openclaw
   ```

2. **기존 설정 백업** (이미 `openclaw onboard` 를 실행했다면)
   ```bash
   cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak
   ```

3. **설정 파일 저장**
   - 아래 예시 JSON 을 `~/.openclaw/openclaw.json` 에 저장한다.
   - `nano ~/.openclaw/openclaw.json` 또는 에디터로 열어 붙여넣기 후 저장한다.

4. **환경에 맞게 수정**

    - `channels.slack.botToken`, `channels.slack.appToken` - Slack 2·3단계에서 발급한 토큰
    - `gateway.auth.token` - 브라우저 확장 프로그램에 입력할 값 (온보딩 시 자동 생성된 것을 그대로 사용하거나 새로 생성)
    - `agents.defaults.workspace` - 워크스페이스 경로 (예: `/home/사용자명/.openclaw/workspace`)
    - `browser.executablePath` - Chrome 설치 경로 (Ubuntu 기본: `/usr/bin/google-chrome`)

5. **Gateway 재시작**
    ```bash
    openclaw gateway restart
    ```

## 8. 개발자 꿀팁: Zod 스키마 참고하기

OpenClaw의 공식문서에 각 설정들에 대한 내용이 있지만 어떤 설정이 있는지를 빠르게 찾기 위해 Zod 스키마를 참고하는 것도 좋다.

OpenClaw는 설정 필드가 많고 복잡한데, CLI 명령으로 다 확인되지 않는 상세 옵션이나 유효한 값을 알고 싶다면 오픈소스 코드를 참고하는 것이 좋다.

**참고 경로**: `src/config/zod-schema.ts`

**방법**

1. [OpenClaw 프로젝트](https://github.com/openclaw/openclaw)를 클론한다.
2. `src/config/zod-schema.ts` 및 관련 `zod-schema.*.ts` 파일을 연다.
3. `gateway`, `channels`, `browser` 등 각 섹션에 어떤 필드가 정의되어 있고, 어떤 타입(string, enum 등)을 허용하는지 확인한다. 커스텀 설정 시 매우 유용하다.

# 결론

가상머신에 OpenClaw를 설치하고 Slack을 연동하면, 격리된 환경에서 AI 에이전트를 안전하게 활용할 수 있다. UTM + Ubuntu VM으로 보안 리스크를 줄이면서도, 브라우저 연동과 다양한 설정으로 Cursor에 버금가는 경험을 제공한다.

초기 설정 단계가 다소 복잡하지만, 온보딩 마법사를 따라 진행하면 순서대로 구축할 수 있다. Slack Pairing, Browser 확장 프로그램 연동, `openclaw.json` 커스터마이징까지 완료하면 슬랙에서 자연어로 PC를 제어하는 에이전트를 사용할 수 있다.
