# Claude Code Statusline 커스터마이징하기

Claude Code CLI를 쓰다 보면 하단에 표시되는 statusline이 꽤 단순하다는 걸 느끼게 된다. Claude Code는 `statusLine.command` 설정을 통해 shell script로 statusline을 완전히 커스터마이징할 수 있다.

이 글에서는 **드라큘라(Dracula) 테마**를 적용한 statusline을 만들어본 과정을 공유한다.

---

## 최종 결과물

![statusline](/images/blog-posts/17-Claude-Code-statusline-커스터마이징하기/statusline.png)

- **Line 1**: 컨텍스트 윈도우 사용률 / 5시간 token rate limit / 7일 token rate limit
- **Line 2**: 모델명 / 프로젝트 경로 / git 브랜치

## 설정 방법

### 1. settings.json 설정

`~/.claude/settings.json`에 다음을 추가한다.

```json
{
  "statusLine": {
    "command": "bash /Users/your-username/.claude/statusline-command.sh"
  }
}
```

### 2. 스크립트 파일 생성

`~/.claude/statusline-command.sh` 파일을 만들고 아래 스크립트를 작성한다.

## 스크립트 구조

### JSON 입력

Claude Code는 statusline 스크립트를 실행할 때 `stdin`으로 JSON 데이터를 전달한다. 주요 필드는 다음과 같다.

| 필드 | 설명 |
|------|------|
| `model.display_name` | 현재 모델명 |
| `workspace.current_dir` | 현재 작업 디렉토리 |
| `context_window.used_percentage` | 컨텍스트 윈도우 사용률 (%) |
| `rate_limits.five_hour.used_percentage` | 5시간 rate limit 사용률 (%) |
| `rate_limits.seven_day.used_percentage` | 7일 rate limit 사용률 (%) |

> `rate_limits` 데이터는 첫 API 응답 이후부터 제공된다. 세션 최초 시작 시에는 플레이스홀더(`──────────`)로 표시된다.

### Dracula 팔레트

터미널 true color(24-bit)를 지원하는 환경에서 Dracula 팔레트를 그대로 사용할 수 있다.

| 색상 | hex | 용도 |
|------|------|------|
| Green | `#50fa7b` | 바 (0~49%) |
| Orange | `#ffb86c` | 바 (50~75%) |
| Red | `#ff5555` | 바 (76%+) |
| Cyan | `#8be9fd` | 프로젝트 경로 |
| Pink | `#ff79c6` | git 브랜치 |
| Purple | `#bd93f9` | 모델명 |
| Foreground | `#f8f8f2` | 텍스트 레이블 |
| Comment | `#6272a4` | 구분자 / 플레이스홀더 |

### 프로그레스 바

10칸짜리 바를 `█`(채움)와 `─`(빈 칸)으로 구성한다. 사용률이 1% 이상이면 최소 1칸은 채워지도록 처리했다.

```shell
make_bar() {
  _pct="$1"
  _filled=$(awk "BEGIN { printf \"%d\", ($_pct / 100) * 10 }")
  [ "$_pct" -gt 0 ] && [ "$_filled" -eq 0 ] && _filled=1
  _empty=$(( 10 - _filled ))
  _bar=""
  _i=0
  while [ "$_i" -lt "$_filled" ]; do _bar="${_bar}█"; _i=$(( _i + 1 )); done
  _i=0
  while [ "$_i" -lt "$_empty" ]; do _bar="${_bar}─"; _i=$(( _i + 1 )); done
  printf '%s' "$_bar"
}
```

### 구분선

`tput cols`로 터미널 너비를 읽어 구분선이 화면에 꽉 차도록 한다.

```shell
_cols=$(tput cols 2>/dev/null || echo 80)
divider="$(printf "${D_COMMENT}%${_cols}s${RESET}" | tr ' ' '─')"
```

### 멀티라인

`printf`로 여러 줄을 출력하면 Claude Code가 멀티라인 statusline을 지원한다. 단, 너무 많은 줄(4줄 이상)은 잘릴 수 있으니 3줄 이하로 유지하는 게 좋다.

```shell
printf "%b\n%b\n%b\n%b\n%b\n" "$divider" "$line1" "$divider" "$line2" "$divider"
```

## 전체 스크립트

```shell
#!/bin/sh
# Claude Code status line — Dracula theme
# Line 1: ctx bar │ 5h bar │ 7d bar
# Line 2: ◆ model ❯ 📁 project ❯ ⎇ branch

input=$(cat)

# --- Extract values from JSON ---
cwd=$(echo "$input" | jq -r '.workspace.current_dir // .cwd // ""')
model=$(echo "$input" | jq -r '.model.display_name // ""')
used_pct=$(echo "$input" | jq -r '.context_window.used_percentage // empty')
five_hour_pct=$(echo "$input" | jq -r '.rate_limits.five_hour.used_percentage // empty')
five_hour_resets=$(echo "$input" | jq -r '.rate_limits.five_hour.resets_at // empty')
seven_day_pct=$(echo "$input" | jq -r '.rate_limits.seven_day.used_percentage // empty')
seven_day_resets=$(echo "$input" | jq -r '.rate_limits.seven_day.resets_at // empty')

# --- Git branch ---
git_branch=""
if git -C "$cwd" rev-parse --git-dir > /dev/null 2>&1; then
  git_branch=$(git -C "$cwd" --no-optional-locks symbolic-ref --short HEAD 2>/dev/null \
    || git -C "$cwd" --no-optional-locks rev-parse --short HEAD 2>/dev/null)
fi

# --- Project name (home-relative path) ---
project_name=""
if [ -n "$cwd" ]; then
  _home="$HOME"
  case "$cwd" in
    "$_home"/*) project_name="~/$(printf '%s' "${cwd#$_home/}")" ;;
    *)          project_name="$cwd" ;;
  esac
fi

# --- Progress bar helper (10 chars) ---
make_bar() {
  _pct="$1"
  _filled=$(awk "BEGIN { printf \"%d\", ($_pct / 100) * 10 }")
  [ "$_pct" -gt 0 ] && [ "$_filled" -eq 0 ] && _filled=1
  _empty=$(( 10 - _filled ))
  _bar=""
  _i=0
  while [ "$_i" -lt "$_filled" ]; do _bar="${_bar}█"; _i=$(( _i + 1 )); done
  _i=0
  while [ "$_i" -lt "$_empty" ]; do _bar="${_bar}─"; _i=$(( _i + 1 )); done
  printf '%s' "$_bar"
}

# --- Dracula palette (true color) ---
D_GREEN='\033[38;2;80;250;123m'       # #50fa7b
D_ORANGE='\033[38;2;255;184;108m'     # #ffb86c
D_RED='\033[38;2;255;85;85m'          # #ff5555
D_CYAN='\033[38;2;139;233;253m'       # #8be9fd
D_PINK='\033[38;2;255;121;198m'       # #ff79c6
D_PURPLE='\033[38;2;189;147;249m'     # #bd93f9
D_FG='\033[38;2;248;248;242m'         # #f8f8f2 foreground (white)
D_COMMENT='\033[38;2;98;114;164m'     # #6272a4 comment (dim purple)
RESET='\033[0m'

SEP="$(printf "${D_COMMENT} ❯ ${RESET}")"
SEP1="$(printf "${D_COMMENT} │ ${RESET}")"

# --- Dynamic color by percentage (Dracula: green/orange/red) ---
pct_color() {
  _pct="$1"
  if [ "$_pct" -gt 75 ]; then
    printf '%s' "$D_RED"
  elif [ "$_pct" -ge 50 ]; then
    printf '%s' "$D_ORANGE"
  else
    printf '%s' "$D_GREEN"
  fi
}

# --- Context bar ---
progress_bar=""
pct_int=""
if [ -n "$used_pct" ]; then
  pct_int=$(printf "%.0f" "$used_pct" 2>/dev/null || echo "$used_pct")
  progress_bar=$(make_bar "$pct_int")
fi

# --- 5-hour rate limit ---
five_hour_bar=""
five_hour_int=""
if [ -n "$five_hour_pct" ]; then
  five_hour_int=$(printf "%.0f" "$five_hour_pct" 2>/dev/null || echo "$five_hour_pct")
  five_hour_bar=$(make_bar "$five_hour_int")
fi

# --- 7-day rate limit ---
seven_day_bar=""
seven_day_int=""
if [ -n "$seven_day_pct" ]; then
  seven_day_int=$(printf "%.0f" "$seven_day_pct" 2>/dev/null || echo "$seven_day_pct")
  seven_day_bar=$(make_bar "$seven_day_int")
fi

line1=""
line2=""

# Line 1: ctx │ 5h │ 7d  (bar=dynamic Dracula color, text=foreground white)
_empty_bar="──────────"
if [ -n "$progress_bar" ]; then
  _color=$(pct_color "$pct_int")
  line1="${line1}$(printf "${D_FG}ctx ${RESET}${_color}%s${RESET}${D_FG} ${pct_int}%%${RESET}" "$progress_bar")"
else
  line1="${line1}$(printf "${D_COMMENT}ctx %s --%${RESET}" "$_empty_bar")"
fi
if [ -n "$five_hour_bar" ]; then
  _color=$(pct_color "$five_hour_int")
  _5h_label="⏱ 5h"
  [ -n "$five_hour_resets" ] && _5h_label="⏱ $(date -r "$five_hour_resets" "+%H:%M" 2>/dev/null)"
  line1="${line1}${SEP1}$(printf "${D_FG}%s ${RESET}${_color}%s${RESET}${D_FG} ${five_hour_int}%% used${RESET}" "$_5h_label" "$five_hour_bar")"
else
  line1="${line1}${SEP1}$(printf "${D_COMMENT}⏱ -- %s --%%  used${RESET}" "$_empty_bar")"
fi
if [ -n "$seven_day_bar" ]; then
  _color=$(pct_color "$seven_day_int")
  _7d_label="📅 7d"
  [ -n "$seven_day_resets" ] && _7d_label="📅 $(date -r "$seven_day_resets" "+%m/%d" 2>/dev/null)"
  line1="${line1}${SEP1}$(printf "${D_FG}%s ${RESET}${_color}%s${RESET}${D_FG} ${seven_day_int}%% used${RESET}" "$_7d_label" "$seven_day_bar")"
else
  line1="${line1}${SEP1}$(printf "${D_COMMENT}📅 -- %s --%%  used${RESET}" "$_empty_bar")"
fi

# Line 2: model ❯ project ❯ branch
_model_display="${model:-loading...}"
line2="$(printf "${D_PURPLE}◆ %s${RESET}" "$_model_display")"
if [ -n "$project_name" ]; then
  line2="${line2}${SEP}$(printf "${D_CYAN}📁 %s${RESET}" "$project_name")"
fi
if [ -n "$git_branch" ]; then
  line2="${line2}${SEP}$(printf "${D_PINK}⎇ %s${RESET}" "$git_branch")"
fi

_cols=$(tput cols 2>/dev/null || echo 80)
divider="$(printf "${D_COMMENT}%${_cols}s${RESET}" | tr ' ' '─')"
printf "%b\n%b\n%b\n%b\n%b\n" "$divider" "$line1" "$divider" "$line2" "$divider"
```

## 주의사항

- **true color 지원 필요**: iTerm2, Kitty, Alacritty 등 24-bit color를 지원하는 터미널에서만 드라큘라 색상이 정확하게 표시된다.
- **jq 필요**: JSON 파싱을 위해 `jq`가 설치되어 있어야 한다. (`brew install jq`)
- **rate_limits 데이터**: 세션 최초 시작 시에는 rate limit 데이터가 없어 플레이스홀더로 표시되다가, 첫 API 응답 이후부터 실제 값으로 업데이트된다.
- **멀티라인 제한**: Claude Code statusline은 5줄 출력 시 잘릴 수 있다. 현재 구성(구분선+line1+구분선+line2+구분선)은 5줄로 Claude Code v2.1.90 기준으로 동작한다.
